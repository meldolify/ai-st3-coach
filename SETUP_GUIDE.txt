STEP-BY-STEP SETUP
===============================================================================

STEP 1: GOOGLE CLOUD SETUP (15 minutes)
----------------------------------------
1. Go to: https://console.cloud.google.com/
2. Click "Create Project" → Name: "ST3-Coach-V4"
3. Select your new project from the dropdown
4. Go to: APIs & Services → Library
5. Search: "Cloud Text-to-Speech API"
6. Click "Enable"
7. Go to: IAM & Admin → Service Accounts
8. Click "Create Service Account"
   - Name: st3-tts-service
   - Click "Create and Continue"
9. Grant role: "Cloud Text-to-Speech User"
10. Click "Done"
11. Click on the service account you just created
12. Go to "Keys" tab → "Add Key" → "Create new key"
13. Choose JSON format → Click "Create"
14. Download the JSON file
15. Rename it to: google-tts-key.json
16. Move it to: AI ST3 Coach V4/backend/

STEP 2: BACKEND SETUP (5 minutes)
----------------------------------
Open terminal in: AI ST3 Coach V4/backend/

# Install dependencies
npm install

# This installs:
# - @google-cloud/text-to-speech (Google TTS)
# - openai (GPT-4o-mini)
# - ws (WebSocket server)
# - dotenv (environment variables)

# Create .env file
cp .env.template .env

# Edit .env file and add your keys:
OPENAI_API_KEY=sk-your-openai-key-here
GOOGLE_APPLICATION_CREDENTIALS=./google-tts-key.json
PORT=8080

STEP 3: START BACKEND (1 minute)
---------------------------------
In backend/ directory:

npm start

You should see:
  ✓ API clients initialized
  ✓ WebSocket server running on port 8080
  ✓ Server ready

If you see errors:
- "OPENAI_API_KEY not found" → Check .env file
- "Google TTS Error" → Check google-tts-key.json path
- "Port already in use" → Change PORT in .env

STEP 4: START FRONTEND (1 minute)
----------------------------------
Open NEW terminal in: AI ST3 Coach V4/frontend/

node serve.js

You should see:
  ✓ Frontend server running at http://localhost:3000/

STEP 5: TEST THE APP (2 minutes)
---------------------------------
1. Open Google Chrome or Microsoft Edge
2. Go to: http://localhost:3000/
3. You should see "ST3 Plastic Surgery Interview Trainer" page
4. Click to expand: Plastic Surgery ST3 Interview → Clinical Stations → Emergencies
5. Click: "Necrotising Fasciitis"
6. You should see:
   - Scenario title
   - Clinical image
   - Status indicators (all "Disconnected" initially)
   - Connect/Disconnect buttons
7. Click "Start Session"
8. Allow microphone access when prompted
9. Wait for "Session ready!" message in log
10. Say: "Hello, I'm ready to begin"
11. AI should respond with audio!

===============================================================================
TROUBLESHOOTING
===============================================================================

ISSUE: "Speech Recognition not supported"
SOLUTION: You must use Chrome or Edge. Safari/Firefox won't work.

ISSUE: Backend won't start - "OPENAI_API_KEY not found"
SOLUTION: 
  1. Check .env file exists in backend/
  2. Check .env contains: OPENAI_API_KEY=sk-...
  3. No spaces around the = sign

ISSUE: "Google TTS Error"
SOLUTION:
  1. Check google-tts-key.json is in backend/ folder
  2. Check .env has correct path: GOOGLE_APPLICATION_CREDENTIALS=./google-tts-key.json
  3. Verify Text-to-Speech API is enabled in Google Cloud Console

ISSUE: Microphone not working
SOLUTION:
  1. Check browser permissions (click lock icon in address bar)
  2. Reload page and allow mic access
  3. Check microphone works in other apps

ISSUE: WebSocket connection failed
SOLUTION:
  1. Backend running? Check terminal with "npm start"
  2. Port 8080 available? Try changing PORT in .env
  3. Firewall blocking? Allow port 8080

ISSUE: AI responds but no audio plays
SOLUTION:
  1. Check browser console for errors (F12)
  2. Volume not muted?
  3. Check backend terminal for Google TTS errors
